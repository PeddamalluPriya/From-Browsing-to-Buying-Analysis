# 🛍️ From Browsing to Buying: $132k Revenue Recovery Analysis

## 📌 Executive Summary
This project analyzes customer friction points across Instagram, Myntra, and Offline stores. By building a custom **Friction Score**, I identified a **$132,010 revenue leakage** caused primarily by slow response times and hidden pricing.

## 🚀 Key Insights
* **The 22-Minute Window:** Response times over 22 minutes correlate with a 60% drop in purchase intent.
* **Shipping Barrier:** Abandonment spikes significantly when shipping costs exceed $80.
* **Solution:** Identified that capping shipping at $45 and automating "DM for Price" queries could recover the lost revenue.

## 📊 Data Source
The dataset used in this analysis is an **original behavioral dataset** created to simulate omnichannel retail journeys. It tracks 5,000+ customer interactions across:
* **Instagram:** High-intent browsing with manual friction (DMs).
* **Myntra:** Low-friction, automated e-commerce checkout.
* **Offline Stores:** Immediate physical conversion.
[View Dataset on Kaggle](https://www.kaggle.com/datasets/priyapeddamallu/browsing-to-buying-shopping-behavior-dataset)
